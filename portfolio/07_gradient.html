Project Gradient Noise

3D Graphics

Project Overview

In this project, I implemented an Improved 3D Perlin Gradient Noise algorithm directly on the GPU using GLSL. To support high-precision floating-point noise maps, I enhanced my texture loading and framebuffer allocation systems in C++ to handle R32F (single-channel 32-bit floating point), RGBA32F, and RGBA8 color formats. The noise values are generated in real-time onto an offscreen framebuffer, and visualized either as a seamless 2D texture slice or as a displaced 3D terrain-like surface. The procedural patterns implemented include Plain Gradient Noise, Fractal Sum, Turbulence, Marble, and Wood.

Tasks Completed

  • Implemented the Improved 3D Perlin Gradient Noise algorithm inside custom GLSL shaders (gen_gradient_noise.vert/.frag).
  • Embedded a hardcoded 64-entry permutation table (duplicated to 128) and implemented Ken Perlin's quintic fade curve ($6t^5 - 15t^4 + 10t^3$) in GLSL.
  • Programmed gradient vector selection from 12 fixed directions cycled by hash values, calculating smooth spatial noise interpolation.
  • Developed GLSL mathematical functions to evaluate Plain Gradient, Fractal Sum, Turbulence, Marble, and Wood patterns directly on the GPU.
  • Enhanced C++ opengl::Texture class with a custom LoadAsFormat function supporting different color formats (RGBA8, RGBA32F, R32F) and incorporating fallback logic for older OpenGL versions using GL::TexStorage2D or GL::TexImage2D.
  • Updated C++ opengl::FrameBuffer specification to support R32F formats for real-time offscreen texture generation.
  • Created vertex (simple.vert, surface.vert) and fragment (simple.frag) shaders to map the generated float textures onto a full-screen quad and to dynamically displace the vertex positions of a plane mesh along the Z-axis to visualize the noise as 3D terrain.
  • Designed interactive UI controls in C++ with ImGui to modify tiling scale, Z-coordinate input, terrain area/height scaling, and mesh stack/slice density in real-time.

Reflection

Developing Perlin Gradient Noise highlighted the performance advantages of GPU-based procedural generation compared to CPU-based lattice sampling. Generating 3D noise in real-time on an offscreen R32F framebuffer allowed me to maintain interactive frame rates even at high resolutions. A major challenge was achieving perfect, seam-free tiling when repeating the texture. By scaling the inputs to the 64-period permutation table and implementing safe coordinate wrapping (safe_mod) in GLSL, I ensured the texture wraps seamlessly. Displacing plane mesh vertices in surface.vert using the noise texture was an exciting challenge, showing how procedural noise can be used to generate realistic, dynamic 3D terrains. This project deepened my understanding of GPU shaders, framebuffer-object management, and advanced graphics programming.
