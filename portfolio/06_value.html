Project Value Noise

3D Graphics

Project Overview

In this project, we implemented a procedural texture generation system in C++ using a Value Noise algorithm. The system supports 1D, 2D, and 3D noise generation on a lattice using a permutation hash, allowing for adjustable interpolation methods to smooth grid values. We integrated the generated textures dynamically into a custom OpenGL pipeline with dedicated shaders, creating high-fidelity visual representations of procedural patterns such as wood, marble, turbulence, and fractal sums.

Tasks Completed

  • Implemented 1D, 2D, and 3D Value Noise template classes in C++ (ValueNoise.hpp).
  • Designed a native compile-time constexpr_floor helper to handle coordinate decomposition, achieving standard C++20 constexpr compatibility under MSVC (NoiseCoordinate.hpp).
  • Programmed multiple smooth fade interpolation parameters including Linear, Cosine, Smoothstep, and Quintic algorithms (Fade.hpp).
  • Implemented standard linear, bilinear, and trilinear spatial mixing algorithms (Interpolation.hpp).
  • Implemented random-shuffled permutation hashing (PermutationHash.hpp) utilizing bitwise masking to enable correct boundary wrapping and seamless tiling when repeated.
  • Enhanced the opengl::Texture wrapper class to support allocating empty GPU textures (LoadAsRGBA) and dynamically uploading newly computed noise pixel data (UploadAsRGBA).
  • Created vertex (simple.vert) and fragment (simple.frag) OpenGL shaders to scale and map the generated textures onto full-screen quads.
  • Integrated full ImGui controls into the demo to dynamically adjust tiling scale, smooth methods, noise dimensions, period dimension sizes, patterns, and source lattice point values.

Reflection

Implementing Value Noise highlighted the elegance and complexity of procedural texture generation. Achieving seamless tiling when repeated was a key focus, requiring a strict understanding of how coordinate boundaries wrap using the permutation mask and bitwise operations. Another interesting challenge was standardizing the C++20 constexpr implementation of coordinate calculations on MSVC, which was resolved by designing a custom native constexpr_floor to avoid compiler dependencies. Decomposing the noise calculation progressively across frames successfully prevented program freezing without relying on multi-threading. The project was incredibly rewarding, resulting in beautifully rendered marble, wood, and turbulence patterns.
